MongoDB는 NoSQL 데이터베이스다.
전통적인 관계형 데이터베이스(SQL)와 달리 스키마가 없고 데이터를  JSON 형태로 저장한다.
문서(document) 기반으로 데이터를 저장하며 컬렉션(collection) 단위로 관리된다.

*여기서 json형태란*
JSON은 JavaScript Object Notation의 약자로, 데이터를 교환하고 저장하기 위한 경량의 텍스트 기반 포맷이다
사람이 읽고 쓰기 쉽고, 기계가 파싱(분석)하고 생성하기 쉬워 웹 애플리케이션에서 서버와 클라이언트 간에 데이터를 주고받을 때 가장 널리 사용하는 것이다

사람이 읽기 쉬움: 텍스트로 되어 있어 개발자가 내용을 쉽게 파악할 수 있다.
데이터 형식: 자바스크립트의 객체 문법과 유사하지만, 자바스크립트에 종속되지 않고 독립적인 데이터 포맷으로 활용됩니다.

JSON의 기본 구조 : JSON은 두 가지 기본 구조로 이루어져 있는데 

객체 (Object)와  배열 (Array)로 이루어져 있다.

우선 객체는 중괄호 {} 로 표현하며, '키(key)'와 '값(value)'의 쌍으로 구성 

배열 대괄호 [] 로 표현하며, 순서가 있는 값들의 목록입니다

moongodb는 nosql을 사용하는데 이떄 일반적인 sql과 nosql의 차이점이란 

sql은 관계형 데이터베이스이다  

정형 데이터에 적합: 데이터의 구조가 명확하고 변경이 거의 없는 경우에 유리한다.
강력한 데이터 일관성: ACID (원자성, 일관성, 고립성, 지속성) 트랜잭션을 지원하여 데이터의 무결성을 보장합다.
확장성: 일반적으로 하나의 서버 성능을 높이는 수직 확장에 유리합다.

nosql은 비관계형 데이터베이스이다

비정형/반정형 데이터에 적합: 소셜 미디어의 게시물, IoT 센서 데이터처럼 구조가 유동적이거나 대용량인 경우에 유리한다
뛰어난 확장성: 여러 대의 저렴한 서버를 추가하여 성능을 높이는 수평 확장이 용이한다
높은 가용성: 분산형 구조를 통해 서버 한 곳에 장애가 발생해도 서비스가 계속될 수 있다.
일관성: **결과적 일관성(Eventually Consistent)**을 추구하여 데이터가 최종적으로는 일관된 상태가 되지만, 실시간으로 완벽한 일관성을 보장하지는 않을 수 있다.
